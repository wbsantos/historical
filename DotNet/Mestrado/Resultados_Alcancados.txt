O experimento teve como objetivo testar duas formas de resolver o problema da densidade, uma utilizando a regra da maioria (regra 232) para reticulado unidimensional de tamanho 64 e contorno periódico, e outra utilizando a regra de Toom com reticulado de 8x8, contorno também periódico. A regra de Toom é também a da maioria, mas com vizinhança em L. Isto é, para vizinhança foi utilizado a própria célula, a célula imediatamente acima desta, e a célula à direita da mesma.

Os estados iniciais testados foram gerados através de números aleatórios não repetidos. Foi utilizado a função Random do .NET com algumas alterações, de modo a gerar um número inteiro de 64 bits, sendo cada bit uma posição no reticulado de teste. Tanto para a regra 232 (reticulado unidimensional), quanto para de Toom (reticulado bidimensional), foram utilizados os mesmos estados iniciais que haviam sido sorteados. A diferença se dá que, para regra de Toom, o reticulado de 64 posições foi "quebrado" em 8 blocos e remontado numa matriz 8x8.

Em suma, para os dois métodos foram utilizados os mesmos estados iniciais, apenas alterando a matriz de entrada para a regra devida.

Ambas as regras foram testadas contra a resposta correta, que foi processada através de métodos clássicos (realmente contando o número de bits ligados em cada estado inicial de teste).
Foi gerado uma população de testes de 1.000.003

Recursos utilizados:
	.NET C#: Os autômatos foram processados através de um algoritmo em .NET de forma paralela (20 threads simultâneas)
	SQL Server: Todos os dados foram salvos numa base SQL Server.
